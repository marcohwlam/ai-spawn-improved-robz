# Group System Design

**Goal:** Replace the loose per-unit ratio spawning with up to two cohesive **groups**
(combined-arms formations) whose composition is still generated by the existing phase
ratio, so each group looks like a historical platoon/company, attacks a shared objective,
backfills its own losses, and upgrades automatically as the phase advances.

**Status:** Approved design (2026-06-28). Replaces the standalone ratio army; the proven
ratio/recharge/fail-cooldown/armor-lead machinery is reused inside the group selector.

---

## Background and decisions

- The ratio army is **disabled as a standalone**; all combat units are now spawned **as
  members of a group**. There is no separate un-grouped ratio spawning.
- A group's composition is **not hand-authored**. It is generated by the existing 4-tier
  ratio (now 5-tier, see below) evaluated against that group's own member counts. As the
  phase advances, the ratio the group fills toward changes, so the group **auto-upgrades**
  (living members stay; replacements come in at the new-phase mix as members die).
- Infantry splits into two sub-tiers, **rifle** and **smg**, with their own ratio.
- The four standalone trickles (neutral-flag capper, MG point-defender, officer keep-alive,
  AT-rifle keep-alive) stay **outside** the group system, unchanged.

**Locked parameters:** group size = 8 · max 2 groups · **the 2nd group is created only once
the 1st is full (8 live members)** · rifle:smg = 3:1 (3:2 when losing) · max 1 elite
infantry per group · groups attack de-conflicted targets (recapture-first, never neutral).

---

## Global Constraints

- Lua 5.1 runtime (engine); offline tooling is Lua 5.5. No `goto`/5.2+ syntax. `luac -p`
  must pass; pure functions covered by offline tests.
- Read-only BotApi: no manpower-balance read; affordability inferred from Spawn failures.
- ~1 Spawn accepted per quant tick; spawns spread across quants.
- `QuantsPerSec = 70`. Phase bands: EARLY 0-180s, MID 180-480s, LATE 480s+.
- Group members are the army; there is no double-counting against a separate ratio pool.

---

## Tier system (5 tiers + aux)

`TierOf(t)` returns one of `heavy | medium | light | rifle | smg`, or `nil` for aux.

```
infantry, not flame, mech=true        -> "light"   (rides a vehicle; existing rule)
infantry, not flame, inf=="smg"       -> "smg"
infantry, not flame, otherwise        -> "rifle"
class HeavyTank                       -> "heavy"
class Tank, recharge >= 550           -> "medium"
class Tank, recharge < 550            -> "light"
class Vehicle                         -> "light"
otherwise (AT, MG, sniper, officer,
  AA, artillery, flamer, mortar)      -> nil  (aux)
```

Per-phase target weights (the ratio each group fills toward):

| Phase | heavy | medium | light | rifle | smg | smg if losing |
|-------|-------|--------|-------|-------|-----|---------------|
| EARLY | -     | -      | 1     | 3     | 1   | 2             |
| MID   | -     | 1      | 2     | 3     | 1   | 2             |
| LATE  | 1     | 1      | 2     | 3     | 1   | 2             |

`armorCap` still gates which tiers a phase allows (EARLY: light; MID: medium; LATE: heavy).
"Losing" = `FlagDeficit() > 0`; it sets the `smg` weight to 2.

Aux is still injected on the existing per-cycle schedule, charged to the group that is
currently being filled (an aux member counts as `nil` tier, does not fill a ratio slot).

## Data flow (group lifecycle)

```
 wave fires (every WaveIntervalNow):
   # create at most one more group, and only when there is room:
   #   - 0 groups            -> create the 1st
   #   - 1 group AND it is full (members == size) -> create the 2nd
   if #Groups == 0
   or (#Groups == 1 and #Groups[1].members == Groups[1].size)
        -> create Group{ size=8, members={}, target=PickGroupTarget(exclude others) }
   for each Group g (spread across quants, within wave budget):
       while #g.members < g.size and budget:
           tier  = DecideTier(phase, countByTier(g.members), enemyHasTanks, eligible)
                   # rifle/smg split + losing smg bump baked into phase targets
           unit  = pick from pool for tier, excluding a 2nd elite if g already has one
           Spawn(unit) -> OnGameSpawn tags squad to g, orders it to g.target
 each quant, per Group g:
   prune dead members ; if #members == 0 -> dissolve g
   if #g.members < g.size (idle) -> backfill one toward the ratio (same selector)
   if g.target not attackable    -> g.target = PickGroupTarget(exclude other group)
   members all attack g.target
 phase change is implicit: DecideTier reads CurrentPhase(), so a persistent group's
   backfill naturally pulls the heavier new-phase mix -> the group "upgrades".
```

## Group data structure

```lua
Context.Groups = {}            -- array, at most 2 live groups
-- each group:
--   { members = { [squadId]=true, ... },  -- live squads belonging to the group
--     size    = 8,                          -- target member count
--     target  = "<flagName>" or nil }       -- shared attack flag
Context.SquadGroup = {}        -- squadId -> index into Context.Groups (membership)
```

Group membership is set when a unit is spawned by the group spawner (a pending-group marker
is consumed in `OnGameSpawn`, mirroring the existing `isCapper`/`PendingCapper` handoff).
Trickle spawns (capper/MG/officer/AT) never set the marker, so they are never members.

`countByTier(members)` counts a group's live members by `TierOf` (heavy/medium/light/rifle/
smg) — the per-group equivalent of the old global `GetFieldCounts`.

## Targeting

`PickGroupTarget(excludeName)` chooses the group's attack flag:

```
candidates = flags whose occupant == enemyTeam       (enemy-held only; never neutral)
             and name ~= excludeName                 (de-conflict from the other group)
if any candidate has a LostStamp -> pick the most-recently-lost (recapture first)
else -> pick the highest GetFlagPriority candidate
return its name, or nil if none (members then fall back to normal GetFlagToCapture)
```

Flag-loss tracking: each quant, for every flag, if it was ours last tick and is now enemy,
record `Context.LostStamp[name] = MatchQuants`. `PrevOwned` holds last-tick ownership.

`CaptureFlag(squad)` routing (membership overrides class):
```
squad in a group   -> attack that group's target (fallback: normal pick if target gone)
elif capper        -> CapperFlagPriority
elif defender      -> DefenderFlagPriority   (trickle MG / AT keep-alive)
else               -> GetFlagPriority
```

## Elite cap

Infantry assault squads tagged `elite=true` (elites/elites_44, stormtroopers*, paratroopers,
shock-elite, etc.). When the group spawner/backfill picks a unit, if the target group already
has one live `elite` member, elite units are excluded from that pick. Cap = 1 elite per group.

---

## Unit tagging (bot.data.lua, data task)

A subagent tags every infantry roster entry and validates against RobZ:
- `inf="rifle"` or `inf="smg"` on each `class=UnitClass.Infantry` (non-flame) squad.
  Rifle = standard rifle squads (riflemans, grenadiers, volksgrens, fusiliers, jagers, ...).
  Smg = SMG/assault squads (smgs, sturmgrens, stormtroopers, shock, ...).
- `elite=true` on elite assault squads (elites, elites_44, stormtroopers*, paratroopers,
  shock-elite, guards-elite, ...).
- mech-tagged infantry keep `mech=true` (they classify as light, not rifle/smg).
Validate: every tagged name still resolves in RobZ (like the prior 288-unit check).

## What is removed / kept / added

**Removed (standalone behavior):** the global per-wave ratio spawn, the per-wave shared
`WaveTarget`/`SquadTarget` (superseded by per-group targets), the global between-wave
ratio backfill (superseded by per-group backfill), and the standalone losing-SMG lead
(`SmgLead`, superseded by the per-phase `smg`-weight bump to 2 when losing).

**Kept and reused:** `GetUnitToSpawn` pool building (affordable + cooled + fail-cooldown +
armorCap + phase lock), `DecideTier` deficit math (extended to 5 tiers + losing smg),
armor-lead within a group's fill, recharge / fail cooldown, the aux per-cycle injection,
phase bands, the four trickles, flag helpers, the offline test harness.

**Added:** `Context.Groups`, `SquadGroup`, `LostStamp`/`PrevOwned`, group manager + spawner,
`PickGroupTarget`, `countByTier`, elite cap, `inf`/`elite` tags, debug logging.

## Debug logging

Every group event prints an `[AISPAWN]` line for later review:
- `GROUP_NEW id=<n> target=<flag>` when a group forms.
- `GROUP_FILL id=<n> tier=<t> try=<unit> ok=<bool> size=<cur>/<max>` per member spawn/backfill.
- `GROUP_UP id=<n> phase=<p>` when a group's fill mix shifts to a new phase.
- `GROUP_TARGET id=<n> target=<flag> reason=<recapture|priority>` on (re)target.
- `GROUP_END id=<n>` when a group dissolves (all members dead).
- The existing `WAVE` line gains `groups=<n>`; trickle lines (CAPPER/DEFENDER/OFFICER/ATRIFLE)
  are unchanged.

## Testing

- `luac -p bot.lua` / `luac -p bot.data.lua` after every change.
- Offline pure-function tests (extend `tests/phase_spec.lua`):
  - `TierOf`: rifle vs smg split, mech->light, elite orthogonal to tier.
  - `DecideTier`: 5-tier deficit picks the right tier from a given group count; losing bumps smg.
  - `PickGroupTarget`: prefers a recently-lost enemy flag; excludes neutral and the other
    group's target.
- In-game review via the GROUP_* debug lines: two groups form, hold de-conflicted targets,
  upgrade across phase boundaries, backfill losses, and never exceed 1 elite each.

---

## Out of scope

- More than two groups, or named/role-specialized groups (the `Context.Groups` array leaves
  room to extend later).
- Reading the real manpower pool (no API).
- Hand-authored historical templates (intentionally replaced by ratio-generated groups).
